Detailed Description

Author
Greg Lyons
Chase Malik
Kevin Rhine

PredPreyCell serves as a default cell for the Wa-Tor simulation (representing water cells) It also serves as a super-class for FishCell and SharkCell, which have similar behavior

Some of the methods, like breed() and updateStateandMove(), are empty, because water does not need to breed or update. FishCell and SharkCell override these methods.

The makeCell() method serves as a sort of mini-factory, which is used within the larger Factory class
